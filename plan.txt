สร้างโปรเจ็กระบบ SaaS สำหรับออกแบบรายงานและเอกสารองค์กร โดยให้บริษัทหลายบริษัทเข้ามาใช้งานระบบได้ แต่ละบริษัทมีข้อมูลแยกจากกันด้วย companyId ส่วนเจ้าของระบบเป็นแอดมินกลางที่สามารถจัดการบริษัท ผู้ใช้ แพ็กเกจ หมวดหมู่เอกสาร เทมเพลตกลาง และตรวจสอบการใช้งานระบบได้

ใช้เทคโนโลยีหลักดังนี้

* Next.js App Router
* TypeScript
* Tailwind CSS
* shadcn/ui
* Prisma ORM
* MongoDB
* NextAuth หรือระบบ JWT Auth
* React Hook Form
* Zod Validation
* Zustand สำหรับ global state
* รองรับ Dark Mode / Light Mode
* รองรับ Responsive Layout ทั้ง Desktop, Tablet และ Mobile

## 1. ภาพรวมของระบบ

ระบบนี้คือแพลตฟอร์มสำหรับสร้าง ออกแบบ จัดเก็บ และส่งออกเอกสารของบริษัท เช่น ใบเสนอราคา ใบแจ้งหนี้ ใบกำกับภาษี สัญญาจ้างงาน รายงานประชุม และเอกสารอื่น ๆ

ระบบต้องเป็น Multi-tenant SaaS โดยมีผู้ใช้งาน 2 กลุ่มใหญ่

### 1.1 System Admin

System Admin คือเจ้าของระบบหรือทีมแอดมินกลาง สามารถจัดการระบบทั้งหมดได้ เช่น

* จัดการบริษัทที่มาใช้ระบบ
* จัดการผู้ใช้ของแต่ละบริษัท
* จัดการแพ็กเกจการใช้งาน
* จัดการหมวดหมู่เอกสาร
* จัดการประเภทเอกสาร
* จัดการเทมเพลตกลาง
* ดูประวัติการใช้งานระบบ
* เปิด/ปิดการใช้งานบริษัท
* ดู Dashboard ภาพรวมของระบบ

### 1.2 Company User

Company User คือผู้ใช้งานของบริษัทที่สมัครใช้ระบบ โดยแต่ละบริษัทจะเห็นเฉพาะข้อมูลของตัวเองเท่านั้น

Role ของ Company User ประกอบด้วย

* OWNER เจ้าของบริษัท
* ADMIN แอดมินบริษัท
* ACCOUNTANT ฝ่ายบัญชี
* FINANCE ฝ่ายการเงิน
* HR ฝ่ายบุคคล
* OPERATION ฝ่ายปฏิบัติการ
* STAFF พนักงานทั่วไป
* VIEWER ผู้ดูข้อมูลอย่างเดียว

### 2.3 เอกสารภาษี

ตัวอย่างเอกสาร

* ใบกำกับภาษี
* ใบกำกับภาษีอย่างย่อ
* ใบเพิ่มหนี้
* ใบลดหนี้

* ภ.พ.30

### 2.4 เอกสารบุคคล

ตัวอย่างเอกสาร

* สัญญาจ้างงาน
* ใบสมัครงาน
* ใบลา
* หนังสือรับรองเงินเดือน
* หนังสือเตือนพนักงาน
* ประวัติพนักงาน

### 2.5 เอกสารการดำเนินงาน

ตัวอย่างเอกสาร

* รายงานประจำวัน

## 17. เงื่อนไขสำคัญ

* ทุกหน้าต้องใช้งานจริงได้
* ทุกปุ่มเพิ่ม แก้ไข ลบ ต้องทำงานจริง
* ทุก Form ต้องมี Validation
* ทุก Table ต้องมี Loading, Empty State และ Pagination
* ทุก API ต้องตรวจสอบ Auth
* ทุก Query ฝั่ง Company ต้องกรอง companyId
* ห้ามใช้ข้อมูล Mock ถ้า API พร้อมแล้ว
* เขียน Code ให้แยก Component ชัดเจน
* ตั้งชื่อไฟล์และตัวแปรให้อ่านง่าย
* รองรับภาษาไทย
* รองรับ Font ภาษาไทยใน PDF
* UI ต้องเรียบร้อย ใช้งานง่าย และเหมาะกับระบบบริษัท
